Usefulness First

A small model is interesting only when it helps with a real task.

Each project starts from a concrete local use case, then asks what training, data, evaluation, and deployment are needed to make the model better than a simple baseline.

Tooling Map

Use the best 2026 tools, then make the evidence inspectable.

Adapt

Fine-tune and adapt models with Transformers, PEFT, TRL, LoRA, QLoRA, rerankers, classifiers, and task-specific training loops.

  • Default path: small base model plus adapter.
  • Strong path: trained component beats a real baseline.
  • Advanced path: DPO, RLVR, or context work with clear metrics.
Understand

Use nanochat as a readable full-stack example of tokenization, pretraining, fine-tuning, evaluation, inference, RL, and local chat.

  • Run tiny experiments to see the whole stack move.
  • Use task files to design evaluation exercises.
  • Borrow the lifecycle discipline for focused student projects.
Automate

Use AI agents for hypotheses, code edits, eval tasks, debugging, and reviews. Autoresearch is one example of a constrained experiment loop.

  • Agent prompts are versioned research artifacts.
  • Experiment ledgers record what changed and why.
  • Humans accept claims only after reviewing evidence.
Run Locally

Package artifacts for local inference with Transformers, llama.cpp, Ollama, MLX, ONNX, or task-specific Python runtimes.

  • Report runtime, memory, latency, and quantization.
  • Evaluate the released artifact, not only the training checkpoint.
  • Make another student able to inspect or run it.
