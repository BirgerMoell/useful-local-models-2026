Useful Local Models

Can we make small local models genuinely useful?

5LN714 Research and Development · HT2026

The Point

Small is not the goal. Useful is.

A small model earns its place when it solves a constrained task, runs locally, and beats a simple baseline.

Why Local?

Local models change the research question.

  • They can run without sending data to an external API.
  • They make latency, memory, and deployment part of the evidence.
  • They force clear task design instead of vague capability claims.

Project Shape

Every project has an evidence chain.

Use caseWhat local problem should this model solve?
BaselineWhat simple system must it beat?
TrainingWhat parameters or components change?
EvaluationWhat held-out evidence supports the claim?
ReleaseCan someone inspect or run the artifact locally?

Possible Projects

These are directions, not assignments. Each student formulates the final research question.

Instruction tuning

Teach a local model a narrow task or domain behavior.

Reranking

Train a local search component for document retrieval.

Tool use

Train structured tool-call behavior and measure validity.

GRPO/RLVR

Use exact rewards for math, code, extraction, or verification.

Context

Test whether longer context is actually useful.

Multilingual

Adapt and evaluate local models by language, not only aggregate score.

Invitation

Pick a task small enough to finish, real enough to matter.

The strongest projects will be modest, runnable, and honest.
