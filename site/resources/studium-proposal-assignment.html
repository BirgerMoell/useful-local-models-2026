Useful Local Models

Studium Assignment Text: Project Proposal

Submit your scientific project proposal as a PDF.

Your proposal should follow the normal course instructions for project proposals

and include the following local-model sections.

Your proposal should define your own research question. You may adapt examples

from the theme material, but the project should not simply repeat a prepared

demo unchanged.

The scientific proposal should be at most 3 pages, excluding references. It

should not be much shorter. Use the course proposal template if one is provided.

Required Proposal Structure

Use these headings unless the course coordinator gives another template:

  • Purpose and aims.
  • Survey of the field.
  • Project description.
  • Time plan and implementation.

Optional headings:

  • Project organization, if relevant.
  • Significance, if you have something concrete to say.
  • Preliminary results, if you have already run a pilot experiment.

For a one-person student project, project organization is usually not needed.

Required Theme-Specific Content

Local Use Case

Describe what the local model is supposed to do and why it is useful to run it

locally rather than through an external API.

State the research question that your project will answer.

Model And Training Plan

Specify:

  • base model or model family;
  • training method;
  • what parameters will be updated;
  • expected compute;
  • fallback plan if the method is too expensive or unstable.

Data Plan

Specify:

  • data sources;
  • license or access conditions;
  • train/dev/test split;
  • possible contamination or leakage risks;
  • whether the data can be redistributed.

Baseline And Evaluation

Specify:

  • baseline system;
  • evaluation dataset;
  • metrics;
  • error analysis plan;
  • significance testing plan, if applicable.

Local Deployment Plan

Specify:

  • expected final artifact format;
  • local runtime;
  • target hardware;
  • how another person will run the model.

Ethical Considerations

Discuss relevant risks such as privacy, copyright, bias, environmental cost,

misuse, and overclaiming.

AI Assistance Statement

If you used AI tools while preparing the project idea, code, data plan, or text,

briefly state what tools were used and how you checked the result. This statement

does not count as a substitute for citations, evidence, or your own analysis.

Grading Criteria

To pass, the proposal must:

  • articulate an understandable project idea;
  • motivate the project using previous research;
  • include one or a few clear research questions;
  • describe a feasible method and dataset plan;
  • include a baseline and evaluation plan;
  • fit the time and compute constraints of the course.

Stronger proposals usually have sharper research questions, better grounding in

related work, clearer evaluation design, and a more realistic fallback plan.

The proposal, popular science abstract, and oral proposal presentation are all

part of the proposal module.
