Useful Local Models

Studium Page: Progress Seminars

Progress seminars check scientific feasibility and technical auditability.

Progress Seminar 1: Data And Baseline

Prepare:

  • draft source registry;
  • train/dev/test split plan;
  • baseline result or baseline implementation plan;
  • one example input and expected output;
  • one risk or blocker.

Progress Seminar 2: Training

Prepare:

  • training configuration;
  • first training run result or training log;
  • baseline vs trained-model comparison if available;
  • failure examples.

Progress Seminar 3: Ethics And Release

Prepare:

  • ethical considerations draft;
  • model card draft;
  • dataset card or data statement draft;
  • local deployment plan;
  • artifact manifest draft.

Progress Seminar 4: Final Analysis

Prepare:

  • main result table;
  • error analysis categories;
  • local demo status;
  • final report outline;
  • remaining risks before first submission.
